## 3. Data Flow Diagrams

### 3.1 User Signup → Profile Creation

```
Browser                    Next.js              Supabase Auth        PostgreSQL
   │                          │                       │                    │
   │── POST /auth/signup ─────►│                       │                    │
   │                          │── signUp() ───────────►│                    │
   │                          │                       │── INSERT auth.users►│
   │                          │                       │                    │
   │                          │                       │   TRIGGER fires ───►│
   │                          │                       │                    │── INSERT profiles
   │                          │                       │                    │   (id, username,
   │                          │                       │                    │    aura=0, xp=0)
   │                          │◄─── session JWT ───────│                    │
   │◄─── redirect /dashboard ─│                       │                    │
```

### 3.2 Focus Session → Penalty Flow

```
Browser (FocusMonitor)         lib/focus-monitor.ts      Supabase RPC
   │                                    │                      │
   │  [Tab hidden / blur / mouseleave]  │                      │
   │──────────────────────────────────►│                      │
   │                                    │── violations.add()   │
   │                                    │── start graceTimer   │
   │                                    │   (10 000 ms)        │
   │  [No recovery within 10s]          │                      │
   │──────────────────────────────────►│                      │
   │                                    │── onPenalty() ──────►│
   │                                    │                      │── UPDATE profiles
   │                                    │                      │   SET aura = aura - 5
   │                                    │                      │── INSERT aura_log
   │◄── Realtime: profile updated ──────────────────────────────│
   │    ViolationOverlay rendered       │                      │
```

### 3.3 Quest Completion → Jury → Resolution

```
Submitter              Supabase Storage    user_quests      Jury Members        RPG Engine
   │                          │               │                  │                  │
   │── upload proof ─────────►│               │                  │                  │
   │◄─ signed URL ────────────│               │                  │                  │

   │   pending_verification   │               │                  │                  │
```

## 4. Component Architecture

### 4.1 AuraRing Component

```
Props:
  aura: number              — current Aura value
  size?: 'sm' | 'md' | 'lg' — renders at 48px / 96px / 180px
  animated?: boolean

Internal state:
  ringState = rpgEngine.auraRingState(aura)
    → { color: string, pulseSpeed: number }

Renders:
  <motion.svg>              — Framer Motion spring on strokeDashoffset
    <circle />              — background track
    <motion.circle />       — animated fill arc (0–100% based on aura tier)
    <motion.circle />       — outer glow ring (Classic mode only)
  <text>                    — numeric Aura value in center
```

### 4.2 FocusTimer Component

```
Props:
  durationMinutes: number
  onComplete: () => void
  onViolation: () => void

Internal:
  phase: 'idle' | 'running' | 'grace' | 'violated' | 'complete'
  secondsLeft: number (useInterval)
  FocusMonitor instance (useFocusMonitor hook)

Renders (phase-dependent):
  'idle'     → Start button
  'running'  → Countdown display + progress ring
  'grace'    → Warning overlay (10s countdown bar)
  'violated' → ViolationOverlay flash + -5 Aura toast
  'complete' → Success animation + ProofUpload trigger
```

## 7. Security Architecture

### 7.1 Threat Model

| Threat | Mitigation |
|--------|-----------|
| Tab-switching to cheat timer | Triple-layer detection (Page Visibility + Blur + MouseLeave) |
| Spoofing penalty API | Server validates `auth.uid()` === `userId` before writing |
| Jury collusion | Randomised pool; minimum Aura threshold; self-voting blocked by unique constraint |
| Old photo as proof | EXIF timestamp validation (Sprint 3); optional live-capture mode |
| Cross-user data access | RLS policies on all tables; service role key never in client bundle |
| JWT theft | httpOnly cookies; short expiry (1hr); auto-refresh via Supabase client |
| Storage bucket access | Private bucket; signed URLs with 1hr TTL; server-generated only |

### 7.2 Content Security Policy

```typescript
// next.config.ts
const securityHeaders = [
  { key: 'X-Content-Type-Options',    value: 'nosniff'       },
  { key: 'X-Frame-Options',           value: 'DENY'          },
  { key: 'Referrer-Policy',           value: 'origin-when-cross-origin' },
  {
    key: 'Content-Security-Policy',
    value: [
      "default-src 'self'",
      "script-src 'self' 'unsafe-inline'",          // Next.js requires unsafe-inline
      `connect-src 'self' ${process.env.NEXT_PUBLIC_SUPABASE_URL} wss://*.supabase.co`,
      "img-src 'self' data: blob: *.supabase.co",
      "style-src 'self' 'unsafe-inline'",
    ].join('; '),
  },
];
```

```
Developer                 GitHub Actions CI          Vercel
   │                             │                     │
   │── git push main ───────────►│                     │
   │                             │── npm test ──────── │
   │                             │── npm run build     │
   │                             │── supabase db push  │  ← Migrations applied
   │                             │── deploy ──────────►│
   │                             │                     │── Edge CDN build
   │                             │                     │── Preview URL generated
   │                             │                     │── Production deploy
   │                             │◄── deploy success ──│
   │◄────────────────────────────│                     │

Supabase (managed cloud):
  ├── Auth (GoTrue)              — handles signup, JWT, refresh
  ├── PostgreSQL + PgBouncer     — connection pooling built-in
  ├── Realtime                   — Phoenix channels over WebSocket
  └── Storage                   — S3-compatible private buckets
```
